CLI-first · OAuth2 · TikTok Publishing

Automate your creator pipeline from the command line.

MoneyFarmers is a pragmatic toolkit for creators and teams who post to TikTok at scale. Authenticate once, queue videos, upload as drafts or Direct Post to authorized profiles, and keep a clean audit trail with retries and rate-limit handling.

See the flow Explore features 
# Login (opens browser for TikTok OAuth)
$ mfarm auth --redirect http://localhost:3099/callback

# Upload as draft
$ mfarm post draft \
    --file clip.mp4 \
    --caption "Boss fight highlights" \
    --tags boss,challenge

# Direct Post (after reviewer approval)
$ mfarm post direct --file clip.mp4 --caption "Speedrun 1:23"

# Pull-by-URL (domain verified)
$ mfarm post draft --pull https://cdn.example.com/video/1234.mp4

Flow

From login to post in three steps.

1) Authenticate

Run mfarm auth. We open your browser to TikTok, you grant access, we capture code via http://localhost:<port>/callback and exchange it for tokens.

2) Prepare

Choose Draft or Direct Post, set caption, hashtags, cover frame, and optional scheduling window (where supported).

3) Publish

Upload the file or provide a verified HTTPS URL. We handle chunked transfer, retries, and report the TikTok video ID.

Security

Security-first by design.

Token handling

Access/refresh tokens are stored encrypted at rest (OS keychain or encrypted keystore). Automatic rotation with grace windows and scope checks.

Least privilege

Only the scopes required for Login Kit + Content Posting. No broad access. Clear revocation path and safe failure modes.

FAQ

Do I need a website for a CLI app?
Yes. TikTok requires an official Website URL plus visible links to Terms of Service and Privacy Policy. This page is designed for that requirement.
Can I use pull_by_url?
Yes, after verifying your domain. Media must be reachable over HTTPS without redirects and remain available until ingest completes.
What about rate limits?
We apply jittered backoff, local queuing, and resumable uploads where applicable. You see clear status and retry counts in logs.
